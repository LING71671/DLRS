# memory-graph-demo — v0.6 offline pipeline walkthrough

A self-contained, fully runnable example that exercises the v0.6 memory atom
+ knowledge-graph pipelines and the descriptor → audit/events.jsonl bridge
(#58) end-to-end **without any hosted API and without downloading any model**.
Reading this README + running `bash run_demo.sh` is the fastest way to see
what DLRS v0.6 produces on top of v0.5.

## What's in this folder

| File | What it is | Pipeline stage that consumes it |
| --- | --- | --- |
| `manifest.json` | DLRS record manifest (schema 0.3.0, fictional subject). | starting point — every pipeline reads `record_id` from here |
| `public_profile.json` | What the public registry would surface. | none, but validates against `schemas/public-profile.schema.json` |
| `run_demo.sh` | Driver script that walks all three pipelines + prints the audit chain. | — |
| `artifacts/raw/text/diary.txt` (gitignored) | A short fictional diary excerpt (~3 paragraphs, ~70 words, contains 4 `[A-Z][a-z]+`-shaped names). Regenerated by `run_demo.sh` on every run. | text pipeline reads it |
| `derived/` (gitignored) | Output of `run_demo.sh`. Regenerated on every run. | — |
| `audit/events.jsonl` (gitignored) | Hash-chained log of `derived_asset_emitted` events emitted by the bridge — one per pipeline. | — |

DLRS is pointer-first, so **no raw text is committed to git**. The placeholder
diary is regenerated deterministically by `run_demo.sh` step 0; the checksum
in `manifest.json` (`d011247d1665...`) is fixed against that generator output,
so a clean checkout + one `bash run_demo.sh` is enough to satisfy the
manifest's hash invariant.

## Quick start (offline, no installs beyond `tools/requirements.txt`)

```bash
# from the repo root, after pip install -r tools/requirements.txt
bash examples/memory-graph-demo/run_demo.sh
```

This runs:

1. **Text cleaning.** Reads `artifacts/raw/text/diary.txt`, applies NFKC
   normalisation + the v0.5 redaction rules, writes
   `derived/text/diary.clean.txt`, `.redactions.json`, and a descriptor.
2. **Memory atoms.** `paragraph` backend (deterministic, dependency-free)
   splits the cleaned text into ~3 atom records and writes
   `derived/memory_atoms/diary.atoms.jsonl` + descriptor.
3. **Knowledge graph.** `regex` backend extracts capitalised name candidates
   (4 nodes: `Alice`, `Bob`, `Charlie`, `David`, plus a few co-mentioned
   places like `Beijing` and `European Commission`) and emits
   `derived/knowledge_graph/diary.{nodes,edges}.jsonl` + a single
   `diary.graph.descriptor.json`.

After it finishes you should see eight files under `derived/` (5 artefacts +
3 descriptors). The descriptor → audit bridge writes one
`derived_asset_emitted` event per descriptor, hash-chained, into
`audit/events.jsonl`. `run_demo.sh` ends by printing those three events:

```
Audit log (one derived_asset_emitted event per descriptor):
  L1: derived_asset_emitted pipeline='text' prev_hash=(genesis)
  L2: derived_asset_emitted pipeline='memory_atoms' prev_hash=sha256:...
  L3: derived_asset_emitted pipeline='knowledge_graph' prev_hash=sha256:...
```

Each descriptor's `audit_event_ref` field points at the line of
`audit/events.jsonl` it corresponds to, e.g.
`audit/events.jsonl#L2` for the memory-atom descriptor.

## Optional spaCy backend for memory atoms

`paragraph` is intentionally dependency-free so the demo runs on a vanilla
VM. If you want the spaCy sentence-segmenter backend instead, install spaCy
and the `en_core_web_sm` model first:

```bash
pip install spacy
python -m spacy download en_core_web_sm
REAL_ATOMS=1 bash examples/memory-graph-demo/run_demo.sh
```

The output schema is identical; only atom granularity changes (sentences vs.
paragraphs).

## Skipping the audit bridge

Every pipeline accepts `--no-audit` for fixture / dry-run invocations that
must not produce an audit log entry. The demo deliberately does NOT pass it
so you can inspect a fully populated chain.

## What this demo intentionally does NOT do

- It does not call any hosted AI API. The v0.6 hosted-API opt-in policy
  gate (#59) refuses by default, and this record has no
  `policy/hosted_api.json`.
- It does not exercise the v0.5 ASR / vectorization / moderation pipelines;
  see `examples/asr-demo` for those.
- It does not include real PII. All four entity names (`Alice`, `Bob`,
  `Charlie`, `David`) and the `European Commission` mention are
  unambiguous fictional placeholders.
